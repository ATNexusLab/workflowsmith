# Growth Marketing

## When to Use

- Structure or audit a product landing page
- Define the information hierarchy for conversion
- Write headlines, CTAs, taglines, or product copy
- Map and optimize the acquisition and activation funnel
- Diagnose why users arrive but do not convert
- Ensure the site or app communicates value before asking for action

## Funnel Frameworks

### AIDA — Persuasion Journey

| Stage | Goal | Design element |
|---|---|---|
| **Awareness** | The user recognizes the problem | Headline + hero visual |
| **Interest** | The user understands the solution | Benefits + how it works |
| **Desire** | The user wants the solution | Social proof + guarantee |
| **Action** | The user converts | Clear, low-risk CTA |

### AARRR — Pirate Metrics

| Stage | Question | Core metric |
|---|---|---|
| **Acquisition** | How do users arrive? | CAC, traffic by channel |
| **Activation** | Did they reach the "aha" moment? | Completed-onboarding rate |
| **Retention** | Do they come back? | DAU/MAU, churn |
| **Referral** | Do they invite others? | NPS, invite rate |
| **Revenue** | Do they pay? | MRR, LTV, trial-to-paid conversion |

> Optimize in this order: Retention → Activation → Acquisition. Keeping users is cheaper than buying more traffic.

## High-Converting Landing Page Structure

```text
┌─────────────────────────────────────────────┐
│  NAV — Logo + secondary CTA (login)         │
├─────────────────────────────────────────────┤
│  HERO (above the fold)                      │
│  Headline: [primary benefit in one line]    │
│  Subheadline: [for whom + how]              │
│  Primary CTA: [verb + benefit]              │
│  Hero visual: [product in real context]     │
├─────────────────────────────────────────────┤
│  SOCIAL PROOF — Logos or proof metrics      │
├─────────────────────────────────────────────┤
│  3 KEY BENEFITS                             │
│  [Icon] Title — one line of benefit copy    │
├─────────────────────────────────────────────┤
│  HOW IT WORKS — 3 simple steps              │
├─────────────────────────────────────────────┤
│  TESTIMONIAL — Real quote with name/photo   │
├─────────────────────────────────────────────┤
│  FAQ — Answers the top 3 objections         │
├─────────────────────────────────────────────┤
│  FINAL CTA — Repeat with risk reduction     │
└─────────────────────────────────────────────┘
```

## Copywriting Framework

### Headlines — Proven Formulas

```text
[Desired outcome] without [main pain]
→ "Manage your projects without spreadsheets or status meetings"

[Action verb] + [specific result] + [time/context]
→ "Launch your store in 15 minutes, without code"

For [persona] who [situation]: [benefit]
→ "For freelancers who hate chasing invoices: automate billing"
```

### CTA Writing — Write the Result, Not the Click

| ❌ Generic | ✅ Benefit-led |
|---|---|
| Submit | Get my free analysis |
| Sign up | Start now — free |
| Buy | Reserve my spot |
| Learn more | See how it works in 2 min |

### Benefits vs. Features

For every feature, keep asking "so what?" until the real user benefit appears.

```text
Feature: "Real-time sync"
→ So what? "Everyone sees changes instantly"
→ So what? "No document version conflicts"
→ Benefit: "The whole team stays on the same page — no 'final version v3' email thread"
```

### Objection Handling

Identify the top 3 objections and answer them in the page structure and copy.

| Objection | Response element |
|---|---|
| "It is expensive" | Cost comparison, ROI framing, free trial |
| "It seems complicated" | "How it works in 3 steps", product demo |
| "I do not trust it" | Real testimonials, customer logos, guarantee |
| "I do not need it now" | Real urgency or the cost of inaction |

## CRO Checklist — Conversion Audit

### Above the Fold

- [ ] Is the value proposition clear within 5 seconds?
- [ ] Can the target user identify themselves in the headline?
- [ ] Is there a visible primary CTA without scrolling?
- [ ] Does the visual show real product usage instead of generic decoration?

### Information Hierarchy

- [ ] Is there one primary action per page?
- [ ] Does the reading flow guide attention toward the CTA?
- [ ] Do benefits appear before features?
- [ ] Does social proof appear before the main CTA?

### Friction and Trust

- [ ] Does the CTA explain what happens next ("no credit card", "cancel anytime")?
- [ ] Is there at least one testimonial with a real name and context?
- [ ] Are core objections answered before the CTA?
- [ ] Are conversion forms free of unnecessary fields?

### Mobile

- [ ] Is the CTA visible and easy to tap on a 375px screen?
- [ ] Is body text readable without zooming (minimum 16px)?
- [ ] Are there no pop-ups that block content on mobile?

## Steps

### 1. Product and User Discovery

Before writing a single line:

- What is the value proposition in one sentence?
- Who is the primary user? Include role, pain, and discovery context.
- What is the primary action the page should drive?
- What are the top 3 objections to purchase or sign-up?

### 2. Audit the Current Funnel

Map the current state with AARRR:

- Where users enter and at what volume
- Where the largest drop-off happens
- What the current activation rate is
- What retained users have in common

### 3. Prioritize by Impact

Use ICE to rank improvements:

```text
ICE Score = Impact × Confidence × Ease (1-10 each)
```

Start with high-impact, high-confidence changes.

### 4. Write and Structure

Apply the copywriting framework and the landing-page structure. Write at least 3 headline options before choosing one. Review from the perspective of a skeptical user — no jargon, no unsupported claims.

### 5. Review and Deliver

Run the CRO checklist before handoff. Define the success metric before launch, for example "increase landing-page sign-up rate from X% to Y%." Capture the baseline before publishing.
